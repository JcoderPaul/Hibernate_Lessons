****** Слушатель событий - Event Listener ******

Событие Event - это объект, описывающий изменение состояния источника (сущности), с которым оно связано.
Примером события, в котором участвует пользователь, являются нажатие кнопки, выбор элемента из списка,
ввод символа с клавиатуры и т.д. В HIBERNATE же пользователь может создать, удалить, сохранить, изменить
сущность и это тоже событие.

Слушатель Listener - это уведомляемый о некотором событии объект. Чтобы слушатель смог реагировать на
определенное событие источника он должен быть им зарегистрирован, т.е. подключен к источнику. Listener
должен реализовывать определенные методы для получения и обработки уведомлений о событии (например,
внесение события и ее содержания в БД).

Listener находится в постоянном ожидании, пока в источнике, в котором он зарегистрирован, не наступит
соответствующее событие, при возникновении которого слушатель получает управление. Также слушателю
передается объект события (источник, сущность), чтобы он смог правильно на него отреагировать. Таким
образом, источник вызывает метод-обработчик события, определенный в классе, являющемся блоком прослушивания.
В качестве блоков прослушивания иногда используют внутренние классы. В этом случае в методе, регистрирующем
блок прослушивания в качестве параметра, используется объект этого внутреннего класса.

После обработки события слушатель возвращает управление. Таким образом, для обработки события вызываются
только те слушатели, которые на него "подписались", т.е. были зарегистрированы источником.

Эти понятия универсальны во многих языках программирования и фактически являются паттерном.

****** Event Listener в Hibernate ******

Что касается события - event, можно сказать, что событие заменило перехватчик - Interceptor, однако оно
добавлено в качестве дополнительной функциональности для перехвата предварительных или пост-необходимых
операций до или после функциональной логики. Архитектура событий Hibernate позволяет работать с конкретными
событиями на уровне сохраняемости (persistence layer). Несколько методов интерфейса Session взаимно связаны
с типом события (event type). Класс org.hibernate.event.spi.EventType содержит полномасштабные определенные
типы событий (event types), которые объявлены как перечисления (ENUMs). Он работает, когда делается запрос
для одного из этих методов, событие генерируется Hibernate Session и передается соответствующему EventListener,
настроенному для этого типа события.

В Hibernate также разрешен пользовательский прослушиватель (Listener),

Список некоторых EventListener:
- AutoFlushEventListener;
- DeleteEventListener;
- PostDeleteEventListener;
- PreDeleteEventListener;
- InitializeCollectionEventListener;
- LoadEventListener;
- MergeEventListener;
- PostInsertEventListener;
- PreInsertEventListener;
- SaveOrUpdateEventListener: save, update, save, update;
